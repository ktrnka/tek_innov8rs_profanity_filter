## Tasks Completed

### Task 1: Single Word Detection - "damn"

**Script:** `detect_damn.py`

**Approach:**
- Simple case-insensitive substring search
- No word boundaries (any occurrence of "damn")

**Results:**
```
Total messages: 53,701
Messages flagged (containing 'damn'): 117 (0.22%)

Correct flags (toxic messages): 52
Incorrect flags (non-toxic messages): 65
```

**Key Findings:**
- **More false positives than true positives** (65 vs 52)
- Precision would be only ~44% (52/117)
- The word "damn" appears in both toxic and non-toxic contexts:
  - Toxic: "damn" (standalone)
  - Clean: "damn he chose a good spot" (expression)
  - Clean: "push the damn hill omg" (frustration)

**Insights:**
- Single word detection without context is unreliable
- Context matters significantly for determining toxicity
- This demonstrates the precision/recall tradeoff mentioned in README

### Task 2: Regex-Based Profanity Detector ✓

**Script:** `regex_detector.py`

**Approach:**
- Regex pattern with word boundaries (`\b`) to match whole words only
- Case-insensitive matching
- Initial word list: 7 words (damn, shit, fuck, ass, bitch, bastard, hell)

**Results:**
```
Total messages: 53,701
Messages flagged: 908 (1.69%)

Confusion Matrix:
  True Positives:  752 (correctly flagged toxic)
  False Positives: 156 (incorrectly flagged clean as toxic)
  True Negatives:  43,341 (correctly identified as clean)
  False Negatives: 9,452 (missed toxic messages!)

Metrics:
  Accuracy:  82.1%
  Precision: 82.8%  ← High! When we flag, we're usually right
  Recall:    7.4%   ← Low! We miss 93% of toxic messages
  F1-score:  0.135
```

**Key Findings:**
- **Precision/Recall Tradeoff Demonstrated:**
  - High precision (82.8%): Flagged messages are usually toxic
  - Low recall (7.4%): Only caught 752 of 10,204 toxic messages
- **Word boundaries help:** Better precision than substring matching
- **Small word list limits recall:** Many toxic messages use words not in our list

**Comparison to Single Word ("damn"):**
- Precision: 44% → 82.8% (improved!)
- True positives: 52 → 752 (15x more toxic messages caught)

### Task 3: Expand Word List ✓

**Scripts:** `analyze_missed.py`, `regex_detector_expanded.py`

**Approach:**
- Data-driven: Analyzed 9,452 false negatives to find common words
- Added 22 words based on frequency in missed toxic messages
- Focused on clear insults (idiot, wtf, stupid, moron, retard, etc.)
- Avoided overly generic words (play, move, kill)

**Expanded Word List (29 words total):**
```
damn, shit, fuck, ass, bitch, bastard, hell, idiot, idiots, wtf,
fucking, useless, stupid, retard, retards, moron, morons, ffs, fck,
stfu, trash, dumb, noob, noobs, bot, bots, camper, campers, camping
```

**Results:**
```
             Original (7)  →  Expanded (29)  →  Change
Recall:      7.4%              46.2%             +389%
Precision:   82.8%             86.6%             +3.8%
F1-score:    0.135             0.603             +347%
Accuracy:    82.1%             88.4%             +6.3%

Toxic caught: 752 / 10,204    4,719 / 10,204    +3,967
```

**Key Finding - No Tradeoff!**
- Usually expanding word list decreases precision (more false positives)
- But our precision IMPROVED (82.8% → 86.6%)
- Why? Data-driven selection chose words that are strongly toxic in gaming context
- Words like "idiot," "moron," "retard," "wtf" are almost always insults

**Limitations:**
- Still miss 54% of toxic messages (5,485 false negatives)
- Shows inherent limits of rule-based approaches
- Many toxic messages use creative insults, misspellings, or context

### Task 4: Test on Reddit Usernames ✓

**Script:** `test_usernames.py`

**Results:**
- Tested on 100,000 Reddit usernames
- Flagged: 19 usernames (0.02% false positive rate)
- Most flagged usernames were actually offensive (e.g., "i-fuck-cats", "1-800-EAT-SHIT")
- Very few true false positives (legitimate bots: "cube-bot", "scorereveal-bot")

**Key Finding:**
- **Word boundaries (`\b`) work excellently!**
- No false positives for "assassin", "glass", "bass", "password" (containing "ass")
- Successfully avoided the "Scunthorpe problem"
